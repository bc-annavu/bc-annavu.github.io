How To...

How do I introduce a new pattern (non project dependent)?

  1. Collaborate with a designer to create a new UI Pattern.
  2. Proposed pattern goes through the design review process.
  3. Approved pattern is documented with rules and specs.
  4. Documented pattern is reviewed with the UI Platform team during the weekly UI work session.
  5. UI Platform team builds the new pattern.
  6. Pattern is published to Pattern Lab.

Iâ€™ve identified a pattern in my app that will be useful to others. How do I promote a pattern (to the Pattern Lab)?

  1. Design creates a new pattern for an DT project.
  2. Proposed pattern goes through the design review process.
  3. Approved pattern is handed over to the DT for development.
  4. DT develops the pattern.
  5. Design documents the pattern with rules and specs to promote the pattern.
  6. Documented pattern is reviewed with the UI Platform team during the weekly UI work session.
  7. UI Platform collaborates with the DT to move the necessary code to PL.
  8. Promoted pattern is published to Pattern Lab.

How do I update a pattern?

  1. Design creates an updated pattern design.
  2. Design audits pattern use and assess the risk to updating the pattern.
  3. Proposed pattern goes through the design review process.
  4. Approved pattern is documented with rules and specs.
  5. Documented pattern is reviewed with the UI Platform team during the weekly UI work session.
  6. UI Platform updates the existing pattern.
  7. Updated pattern is published to Pattern Lab.

How can design promote a pattern?

  1. Identify
    During design phase use Pattern Lab to search for component or pattern that fits the needs of your product. If a component or pattern cannot be identified then move towards creating a solution that can be used and foresee any products in the future who can benefit from this. Collaborate and communicate with your fellow designers and engineers to get their perspective and feedback.
  2. Document
    Once a component or pattern has been identified, provide documentation including:
    • Description
    • Usage rules
    • Subcomponents (If applicable)
    • Accessibility rules
    • Wireframes/Screenshots
    • Mobile/Responsive behaviour
  3. Implement
    • Review with Design and UI teams
    • Resourced, tested, released by UI team
